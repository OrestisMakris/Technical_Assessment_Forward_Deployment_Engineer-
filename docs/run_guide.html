02
Set environment variables
API keys and agent ID — required before anything runs
✓
GOOGLE_API_KEYrequired aistudio.google.com → API Keys
ELEVENLABS_API_KEYrequired elevenlabs.io → Profile → API Key
ELEVENLABS_AGENT_IDrequired ElevenLabs → Conversational AI → your agent ID
MAX_ITERATIONSoptional default: 5
PASS_THRESHOLDoptional default: 8.0
DB_PATHoptional default: techmellon.db (project root)
export GOOGLE_API_KEY=AIzaSy... export ELEVENLABS_API_KEY=sk_... export ELEVENLABS_AGENT_ID=agent_... # optional overrides for the walkthrough demo: export MAX_ITERATIONS=3 export PASS_THRESHOLD=7.0
If ELEVENLABS_AGENT_ID is not set, the system falls back to Gemini acting as the agent directly. The loop still runs — useful for testing without ElevenLabs access.
For ElevenLabs: create an agent at elevenlabs.io → Conversational AI → New Agent. Set the webhook URL to http://your-host:8000/webhook/elevenlabs and add the tools listed in the system prompt.

04
Add static file serving
One-line addition to backend/main.py
✓
The generated main.py is missing this one line — add it to serve the UI from the API server.
# in backend/main.py, after all app.include_router() calls, add: from fastapi.staticfiles import StaticFiles app.mount("/", StaticFiles(directory="backend/static", html=True), name="static")
This mounts index.html at / so http://localhost:8000 opens the observer UI directly. The API routes (/flights, /bookings, etc.) still work because they're registered first.
05
Start the server
Seeds the DB automatically on first boot
✓
cd techmellon-fde uvicorn backend.main:app --reload --port 8000
On first boot you'll see: "Database initialised." — this seeds 35 flights and 5 fixed bookings (the ones referenced in the assessment scenarios).
The --reload flag means uvicorn watches for file changes. When the fixer patches a Python file, it sends SIGHUP to reload without a full restart.
# verify the API is up: curl http://localhost:8000/ # → {"service":"TechMellon Airlines API","docs":"/docs","loop_stream":"/loop/stream"} curl http://localhost:8000/flights/search?destination=Tokyo # → list of available Tokyo flights
06
Open the observer UI
The Scuffers-inspired loop dashboard
✓
open http://localhost:8000
You'll see the black ticker, the SIMULATE / EVALUATE / FIX / PUSH step tabs, and three panels. Everything is idle until you click START LOOP.
07
Run the loop
Via UI button or curl
✓
Option A — click START LOOP in the UI. The loop runs in the background; the UI streams live via SSE.
# Option B — curl (all 10 scenarios): curl -X POST http://localhost:8000/loop/start # Option C — specific scenarios only: curl -X POST http://localhost:8000/loop/start \ -H "Content-Type: application/json" \ -d '["book_next_available","cancel_refund","pet_policy"]' # Option D — CLI directly (bypasses FastAPI): python -m refinement_loop.loop --max-iterations 3 --pass-threshold 7.0
The loop runs all 10 scenarios per iteration. With ElevenLabs WS + Claude evals, each iteration takes ~3–5 minutes. In fallback mode (no ElevenLabs), ~1–2 minutes.
Logs are written to logs/run_YYYYMMDD_HHMMSS.json automatically after each run. The final system prompt is saved to prompts/system_prompt.txt.
